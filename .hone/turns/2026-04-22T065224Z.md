## Delta proposed
Final ADR-004 item: scribe_suggest(repo, change_description). Requires delegation to claude -p — scribe itself doesn't import Anthropic SDK (ADR-004's subscription-tooling-only clause). Design: spawn `claude -p` as a subprocess, feed it a prompt that contains the change description + the scribe.yaml tracked list + recent commit context, parse the ranked output. This is the hardest sub-feature because it involves (a) prompt engineering, (b) subprocess management, (c) error handling for missing claude binary / network failures. Should probably start with a well-scoped prompt template and a test that mocks the subprocess call.
